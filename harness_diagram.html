🏗️ Agent Evaluation Harness Architecture

High-Level Pipeline

Your Agent
↓
Adapter
↓
Runner
↓
Metric Layers
↓
Scorecard
↓
CSV/JSON

Core Components

1. Adapter (harness/adapter.py)

Purpose: Normalize any agent flavor to AgentCallable

Agent Type Input Output
Plain callable fn(query: str) → str | dict | AgentResult ✅ Today
LangChain Runnable .invoke(input) 🔜 Sprint 1
LangGraph Graph .invoke(state) 🔜 Sprint 1

2. Models (harness/models.py)

The data contracts that flow through the harness:

Class Purpose Key Fields
EvalCase Test input id, question, ground_truth, reference_contexts
AgentResult Agent output answer, retrieved_contexts, trajectory, latency_s, error
Scorecard Evaluation results per_case, aggregate, metadata
StepRecord One agent action step_type, name, args, output

3. Runner (harness/runner.py)

Purpose: Execute agent over dataset, isolate exceptions, build scorecard

1
Initialize: Wrap agent with adapt(), set up config
2
For each case: Call agent.run(case.question)
3
Isolate: Catch exceptions, record errors, continue to next case
4
Collect: Build DataFrame with [id, question, latency_s, error, ...]
5
Return: Scorecard ready for metric layers

4. Config (harness/config.py)

Purpose: Centralize run configuration

Field Default Purpose
model "gpt-4o-mini" Agent's LLM
eval_model "gpt-4o-mini" Evaluator's LLM (for metrics)
metrics [] Which metrics to compute
dataset_path None Where to load test cases
thresholds {} Pass/fail cutoffs

5. Metric Layers (metrics/)

Purpose: Compute evaluations on the scorecard

Each metric layer reads Scorecard and adds columns to per_case:

  • RAGAS: answer_relevancy, faithfulness, context_precision, context_recall
  • DeepEval: guardrails, bias, hallucination
  • Custom: Tool accuracy, trajectory quality

Data Flow Example

INPUT: EvalCase
id: "q1" | question: "What is ML?" | ground_truth: "ML is a subset of AI"
↓ adapt() + measure latency ↓
CALL: agent.run(query)
Returns: "Machine learning is..." in 0.234s
↓ coerce to AgentResult ↓
OUTPUT: AgentResult
answer: "Machine learning is..." | latency_s: 0.234 | error: None
↓ collect into Scorecard ↓
SCORECARD (before metrics)
per_case DataFrame: [id, question, latency_s, error]
metadata: {n_cases: 50, n_errors: 0}
↓ compute_metrics_mock() ↓
SCORECARD (after metrics)
per_case: [..., answer_relevancy: 0.82, faithfulness: 0.91, ...]
aggregate: {answer_relevancy: 0.785, faithfulness: 0.895}

Exception Isolation

One bad case doesn't abort the run:

for case in cases:
  try:
    result = agent.run(case.question)
  except Exception as e:
    result = AgentResult(answer="", error=str(e))
    # Continue to next case!

Result: Bad cases get error set, metrics skip them (NaN), you see all results.

Key Design Principles

  • Single responsibility: Each component does one thing well
  • Data contracts: Models are the language (not dict keys)
  • Fault isolation: One bad case ≠ bad run
  • Composable metrics: Add layers without changing runner
  • Type safety: Dataclasses → IDE autocomplete, self-documenting

Code Example

from agent_eval_suite import EvalRunner, load_dataset, EvalConfig
from agent_eval_suite.metrics.ragas_simple import compute_metrics_mock

# 1. Define agent (any callable)
def my_agent(query: str) -> str:
    return "The answer is..."

# 2. Load dataset
cases = load_dataset("datasets/wiki_50.json")

# 3. Configure
config = EvalConfig(metrics=["answer_relevancy", "faithfulness"])

# 4. Run
runner = EvalRunner(my_agent, config)
scorecard = runner.run(cases)

# 5. Apply metrics
scorecard = compute_metrics_mock(scorecard, cases, config)

# 6. Export
scorecard.to_csv("results.csv")
scorecard.to_json("results.json")
